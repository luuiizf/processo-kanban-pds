Introdução

Kanban é um método ágil para visualizar o trabalho, limitar WIP e melhorar o fluxo. No PDS Corporativo buscamos previsibilidade, menor lead time, qualidade e transparência.

Aqui você encontra o diagrama do processo, as etapas, os papéis e as práticas usadas.

Fluxo definido: Backlog → A Fazer → Em Andamento → Em Teste → Concluído

Diagrama de atividades

Políticas do fluxo (resumo)
  • Backlog: priorização por valor, critérios rascunhados.
  • A Fazer: DoR simples; puxar por capacidade.
  • Em Andamento: WIP 3; PRs pequenos; pairing quando útil.
  • Em Teste: WIP 2; testes auto e explorações.
  • Concluído: DoD claro; registros essenciais.

    Tabela de atividades

    Etapas do processo e descrições
    Etapa Objetivo Entradas Atividades Saídas Políticas / WIP Métricas
    Replenishment / Planejamento Escolher o que entra no próximo ciclo. Backlog bruto, capacidade, riscos. Revisar, fatiar, priorizar, alinhar. Itens claros para “A Fazer”. Política de seleção visível. Tempo de reunião, % itens prontos.
    Backlog Guardar e ordenar ideias por valor. Ideias, feedbacks. Refino leve, dividir épicos, crit. rascunhados. Itens prontos para “A Fazer”. Sem WIP. Revisão semanal. Age do backlog, % com critérios.
    A Fazer Deixar pronto para começar. Itens priorizados com DoR. Tirar dúvidas, alinhar técnico, estimar leve. Itens aptos ao dev. WIP 5; puxar por capacidade. Bloqueios na entrada.
    Em Andamento Construir com qualidade. Item “A Fazer”. Dev, testes unit., PR e revisão. Pronto para teste. WIP 3; PR pequeno. Cycle (dev), retrabalho.
    Em Teste Validar critérios e regressão. Build candidato. Auto + exploração + evidências. Apto para produção. WIP 2; automação. Defeitos, tempo de teste.
    Implantar Levar para prod com segurança. Build aprovado. Deploy, smoke test, plano de rollback. Versão no ar. Janela/rota de release. Falhas de deploy, MTTR.
    Feedback do Usuário Ouvir e ajustar. Uso real, comentários. Coletar, taguear, decidir ação. Entradas para backlog. Classe de serviço p/ urgentes. Tempo de resposta.
    Gestão de Métricas / Fluxo Controlar o sistema. Dados do quadro/CI. CFD, lead/cycle, throughput. Insights e ações. Revisão semanal. Trend de lead time.
    Review / Demo e Retro Aprender e melhorar. Entrega feita, métricas. Mostrar, ouvir, decidir mudanças. Plano de melhorias. Foco em políticas do fluxo. Ações concluídas.
    Concluído Registrar entrega. Item validado. Doc mínima, changelog, aviso. Item entregue/homologado. DoD: testes verdes, doc. Lead time, throughput.

    Práticas ágeis (geral)

    • Limites de WIP
    • Pull por capacidade
    • DoR e DoD
    • Testes auto (unit, integração, smoke)
    • PR pequeno + review
    • Pair/Mob quando útil
    • Métricas: lead/cycle, throughput, CFD
    • Gestão de bloqueios
    • Classes de serviço (Expedite, Padrão, Data fixa)
    • Retros com ações simples

    Práticas por atividade

    Atividade Práticas
    Replenishment Priorizar por valor; fatiar; definir DoR; limite de itens selecionados
    A Fazer Checklist DoR; evitar dependências; estimativa leve; política de pull
    Em Andamento WIP 3; branch curta; testes unit.; PR pequeno; pair em tarefas críticas
    Em Teste Teste auto (unit/integr.); exploração; evidências; corrigir rápido
    Implantar CI/CD; smoke pós-deploy; feature toggle; plano de rollback
    Feedback Coleta simples; tag por tema; decisões rápidas (volta ao backlog)
    Métricas CFD semanal; lead/cycle; throughput; olhar gargalos
    Review/Retro Demo curta; 2-3 ações de melhoria; revisar políticas do quadro
